# danceisasport.com

A Gen-Z-native dancer profile + booking marketplace.

## Stack
- Next.js 14 (App Router) + TypeScript
- Tailwind CSS (dark mode via `html.dark` + CSS variables)
- Supabase (Postgres + RLS + Auth) via `@supabase/supabase-js` and `@supabase/ssr`
- Cloudinary for video/photo uploads (unsigned, upload preset)
- Framer Motion for animations
- Recharts for analytics charts

## Routes
- `/` — landing page (hero, marquee, how-it-works, featured profiles, merch CTA, video preview, footer)
- `/[handle]` — public dancer profile (e.g. `/jasmine.dances`) — server component, `dynamic = 'force-dynamic'`, logs a `profile_views` row per visit
- `/feed` — TikTok-style vertical video feed (snap-scroll, IntersectionObserver autoplay, infinite scroll, mini booking modal)
- `/onboard` — 4-step dancer wizard (identity, styles, services, media + NIL). Sets `dancers.user_id` if a session exists and routes to `/dashboard`; otherwise routes to `/[handle]`.
- `/login` — magic-link auth (`signInWithOtp`)
- `/auth/callback` — exchanges the magic-link code for a session
- `/dashboard` — auth-gated dancer dashboard with 5 tabs (Profile, Bookings, Merch QR, Analytics, Settings)
- `/admin` — admin-only panel (role from `user_metadata.role === 'admin'`) with 4 tabs (Overview, Dancers, Bookings + CSV, Settings)
- `/offline` — service-worker offline shell
- `/sitemap.xml` — generated from published dancers (`app/sitemap.ts`)
- `/robots.txt` — generated by `app/robots.ts`

## Middleware
`middleware.ts` runs on `/dashboard/:path*`, `/admin/:path*`, `/login`, `/auth/:path*`. It refreshes the session via `@supabase/ssr`, redirects unauthenticated visitors to `/login`, and gates `/admin` on the admin role.

## Environment variables
- `NEXT_PUBLIC_SUPABASE_URL` (URL, public)
- `NEXT_PUBLIC_SUPABASE_ANON_KEY` (publishable, public — **do not mark Sensitive in Vercel**)
- `NEXT_PUBLIC_CLOUDINARY_CLOUD_NAME`
- `NEXT_PUBLIC_CLOUDINARY_UPLOAD_PRESET`

Important: `NEXT_PUBLIC_*` env vars are inlined at build time. Vercel "Sensitive" env vars are not available at build time, so marking them Sensitive will produce empty values in the client bundle and 500s at runtime.

## Supabase tables
- `dancers` — profile + `user_id uuid references auth.users(id)`
- `dancer_styles`, `dancer_services`, `dancer_media`, `dancer_reviews` — dancer-scoped child rows
- `booking_requests` — anonymous inbound booking requests
- `profile_views` — analytics; SHA-256 hashed visitor IP + referrer
- `dancer_likes` — for feed likes (uniqueness on `(media_id, user_id)`)
- `admin_settings` — single-row platform toggles (open_signups, feed_enabled, notify_admin_email)

## RLS summary
- Public read on `dancers WHERE is_published = true` and child rows
- Owner read on own unpublished `dancers`; owner update on own row
- Public insert on `booking_requests`
- Anon insert allowed on `dancers`, `dancer_styles`, `dancer_services`, `dancer_media` for MVP onboarding
- `profile_views`: anonymous insert; only the dancer reads their own
- `dancer_likes`: public read; authenticated user can insert/delete their own
- `admin_settings`: public read; admin-only update (`auth.jwt() -> user_metadata.role == 'admin'`)
- RPC `check_handle_available(p_handle text) RETURNS boolean` — security definer

## Supabase clients
- `lib/supabase.ts` — `serverSupabase()` / `browserSupabase()` (no auth helpers). Used by the public profile + landing data fetches.
- `lib/supabase-server.ts` — `getSupabaseServer()` using `@supabase/ssr` + cookies. Reads the auth session in server components and route handlers.
- `lib/supabase-browser.ts` — `getSupabaseBrowser()` using `@supabase/ssr`. Reads the auth session client-side (single cached instance).

## Cloudinary
Browser uploads to `https://api.cloudinary.com/v1_1/<CLOUD_NAME>/<image|video>/upload` with `upload_preset`. The returned `secure_url` is stored in `dancer_media.url` or `dancers.avatar_url`.

## Design system
- Primary `#7c5cbf` (purple), accent `#5bbfa0` (mint)
- Light bgs `#faf9fc / #f3f1f8 / #ede9f5`; dark bgs `#0f0d14 / #16121f / #1a1625`
- Text `#2a2535 / #5a5468` (light) → `#e8e4f0 / #c4bcd4` (dark)
- Fonts: Cormorant Garamond (headings + bio), Jost (UI)
- Animations: particles, scroll reveals, count-up stats, shimmer, dot pulse, orb drift, spring modals
- Universal: `active:scale-95` micro-press on every CTA

## UI primitives
- `components/ui/Toaster.tsx` — global toast provider + `useToast()` hook. Wrapped around children in root layout. Types: success (mint), error (red), info (purple). Auto-dismiss 3s. Bottom-center on mobile, top-right on desktop.
- `components/ui/PageTransition.tsx` — Framer Motion `AnimatePresence` keyed by pathname, fade + slight Y slide.
- `components/ui/ThemeManager.tsx` — applies `dark` class to `html` based on `localStorage('dia-theme')` falling back to `prefers-color-scheme`. `toggleTheme()` persists choice.
- `components/ui/ThemeToggle.tsx` — sun/moon button in Nav.
- `components/ui/InstallPrompt.tsx` — listens for `beforeinstallprompt`, shows custom A2HS sheet on the 2nd+ visit (tracked via `localStorage('dia-visits')`).
- `components/ui/SwRegister.tsx` — registers `/sw.js` in production after a 1.5s defer.

## PWA
- `public/manifest.json`, `public/icon-{192,512}.svg`, `public/sw.js`
- Manifest linked from `metadata.manifest` in the root layout
- `viewport.themeColor` switches by `prefers-color-scheme`
- Service worker caches the app shell, skips Supabase / Cloudinary / QR API requests, falls back to `/offline`

## SEO
- `metadataBase`, default + per-route `<title>`, OG tags, Twitter card in root layout
- `generateMetadata` on `/[handle]` uses the dancer's name, bio excerpt, and avatar
- `app/sitemap.ts` lists static routes + every published dancer handle
- `app/robots.ts` disallows `/dashboard`, `/admin`, `/auth`, `/login`

## Conventions
- All Supabase calls go through one of the three clients above.
- Hand-written interfaces in `lib/database.types.ts` (the project shares a Supabase instance with other apps; auto-generated types would be ~120k chars).
- Server pages that read or write Supabase are marked `export const dynamic = 'force-dynamic'`.
- Onboarding/dashboard pages are client-leaning shells fed initial data from a server component (`page.tsx` server, `Shell.tsx` client).
- Mobile-first everywhere. Profile + dashboard switch to a single column under 768px; dashboard adds a fixed bottom nav.
